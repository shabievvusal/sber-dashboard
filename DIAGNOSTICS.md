# Диагностика проблем с отчетностью

## Проверка наличия данных

```bash
# 1. Проверьте, есть ли загруженные данные
docker compose exec analyz ls -la /app/data_days

# 2. Проверьте, что маршрут /days возвращает данные
docker compose exec backend curl http://analyz:5050/days

# 3. Проверьте через прокси
docker compose exec backend curl http://localhost:5000/integrations/analyz/days

# 4. Проверьте маршрут /day_summary (замените DATE на реальную дату)
docker compose exec backend curl http://analyz:5050/day_summary/2025-01-19

# 5. Проверьте логи Analyz на ошибки
docker compose logs analyz | grep -i error | tail -20
```

## Возможные причины

1. **Нет загруженных данных** - нужно загрузить файл через интерфейс
2. **Ошибки при обработке данных** - проверьте логи Analyz
3. **Проблемы с проксированием** - проверьте логи backend
4. **Проблемы с правами доступа** - проверьте права на директорию data_days

## Решение

Если данных нет:
1. Загрузите файл через интерфейс загрузки отчетов
2. Дождитесь завершения обработки
3. Обновите страницу

Если данные есть, но не отображаются:
1. Проверьте консоль браузера (F12) на ошибки
2. Проверьте логи контейнеров
3. Убедитесь, что frontend контейнер пересобран с новой конфигурацией nginx
